# Security Policy

## Supported versions

Qlass is pre-1.0 alpha software. Security fixes are applied to the default branch until versioned releases begin.

## Reporting a vulnerability

Please do not open a public issue for vulnerabilities involving authentication, authorization, student data, file access, OAuth tokens, or AI provider keys.

Report privately to the repository owner by email or GitHub private vulnerability reporting if enabled.

## Classroom data principles

- Collect the minimum data needed for the classroom workflow.
- Keep AI provider keys encrypted at rest and never return them to the client.
- Keep AI-generated classwork behind teacher review before students can see it.
- Treat imports, submissions, grades, and attachments as sensitive educational records.
